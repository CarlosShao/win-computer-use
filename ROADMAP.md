# ROADMAP.md - win-computer-use 开发路线图

> 本文档跟踪项目的长期规划功能，按优先级和复杂度排序。
> 更新时间：2026-06-27

---

## 当前版本

**v1.0.0** — 基础功能完整，支持截图、鼠标键盘、窗口管理、UI Automation、图像匹配、OCR。

---

## Phase 1 ✅ 已完成（2026-06-26 ~ 2026-06-27）

| 任务 | 状态 | 说明 |
|------|--------|------|
| README 标注 Windows Only | ✅ | 添加 `[!IMPORTANT]` 警告块 |
| 统一 JSON 输出格式 | ✅ | `{ok, action, data/error}` |
| 接入 RapidOCR | ✅ | 默认后端，Tesseract 可选 |
| 多尺度模板匹配 | ✅ | 支持 `[1.0, 1.25, 1.5]` DPI 缩放 |
| `smart_click()` 降级路由 | ✅ | UI Automation → OCR → 图像匹配 |

---

## Phase 2 🔵 进行中

| 任务 | 优先级 | 复杂度 | 说明 |
|------|----------|--------|------|
| **MCP Server / FastAPI 常驻进程** | 🔴 高 | 🔴 高 | 把 CLI 改成常驻服务，避免每次冷启动 Python 进程，模型只加载一次，响应从秒级降到毫秒级 |
| **`--with-markers` 语义截图** | 🟡 中 | 🔴 高 | 用 OpenCV 检测可点击区域 + 画编号标记，类似 OmniParser（需要研究目标检测算法）|
| **Windows Sandbox 模式** | 🟢 低 | 🟡 中 | 可选在 Windows Sandbox 里运行任务，实现输入隔离 |
| **后台点击（不抢鼠标）** | 🟡 中 | 🟢 低 | 用 `pywinauto` 的 `click_input()` 发后台事件 |

---

## Phase 3 🔵 待规划

| 功能 | 优先级 | 说明 | 依赖 |
|--------|----------|------|--------|
| **可视化工作流编辑器** | 🟢 低 | 拖拽节点生成自动化脚本 | Phase 2 完成后 |
| **自然语言 → 自动化脚本** | 🟡 中 | AI 直接理解"打开浏览器搜XXX"并执行 | LLM API 集成 |
| **录制回放（Macro Recorder）** | 🟡 中 | 录制鼠标键盘操作，保存为脚本可回放 | 无 |
| **跨平台支持（Wayland/Linux）** | 🟢 低 | 研究 Linux 桌面自动化方案 | 需求驱动 |

---

## 技术债务

| 问题 | 严重程度 | 修复建议 |
|--------|------------|----------|
| `scripts/` 模块间耦合较高 | 🟡 中 | 抽离公共接口层 |
| 没有单元测试 | 🟡 中 | 用 `unittest` 或 `pytest` 补测试 |
| `click-image` 没做 NMS（非极大值抑制）| 🟢 低 | 多结果时坐标重叠，需去重 |
| OCR 结果没做缓存 | 🟢 低 | 同一张图多次 OCR 浪费算力 |
