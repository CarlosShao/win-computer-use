## 设计原则

本技能作为 WorkBuddy 的轻量级桌面自动化 Skill，遵循以下原则：

1. **轻量优先**：不依赖 Docker / WSL2 / 虚拟机等重型组件，普通 Windows 用户开箱即用
2. **操作透明**：所有操作必须对用户可见，不能出现"不知道 AI 干了什么"的情况
3. **性能优先**：响应快、资源占用低，常驻进程模式下 OCR 模型只加载一次
4. **诚实设计**：不假装"隔离"，操作前明确告知用户，操作中显示进度
5. **安全可控**：敏感操作通过 `safety.py` 拦截并提示用户确认

> 📌 **实测结论（2026-06-27）**：市面产品（Hermes、Codex）均无真正沙箱隔离，
> 所谓"独立光标"只是视觉反馈，底层仍操作真实屏幕，用户随时可干预。
> 因此本技能不追求"假隔离"，采用"诚实告知 + 可选鼠标锁"方案。

> 核心目标：**让技能更好用、更智能**
> 所有功能均以"轻量、无额外依赖"为前提。

| 功能 | 优先级 | 说明 | 依赖 / 备注 |
|--------|----------|------|-------------|
| **录制回放（Macro Recorder）** | 🔴 高 | 点击"开始录制"，用户手动操作一遍，自动生成可回放的自动化脚本。零学习成本。 | 无额外依赖 |
| **自然语言 → 自动化脚本** | 🟡 中 | 用户用自然语言描述任务，AI 自动生成可执行的自动化脚本。 | ⚠️ 更适合集成在 **WorkBuddy 平台层**，技能只提供底层能力 |
| **可视化工作流编辑器** | 🟢 低 | 将操作过程变成可视化流程图，支持查看/回放/修改/导出。 | 需要 GUI；可独立开发 |
| **沙箱模式（可选高级功能）** | ⬜ 不考虑 | 市面产品均无真正沙箱隔离，且需要 Docker+WSL2 高门槛，普通用户无法使用。**正式从路线图中移除。** | — |
| **跨平台支持** | ⬜ 不考虑 | 项目定位为 Windows 专属。 | — |

## 技术债务

| 问题 | 严重程度 | 修复建议 |
|--------|------------|----------|
| `scripts/` 模块间耦合较高 | 🟡 中 | 抽离公共接口层 |
| 没有单元测试 | 🟡 中 | 用 `unittest` 或 `pytest` 补测试 |
| `click-image` 没做 NMS（非极大值抑制）| 🟢 低 | 多结果时坐标重叠，需去重 |
| OCR 结果没做缓存 | 🟢 低 | 同一张图多次 OCR 浪费算力 |
